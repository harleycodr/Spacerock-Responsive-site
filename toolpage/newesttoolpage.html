    • Cron Tab
    Click to hide.
    * * * * * /home/sites/sitexx/users/es55xxx/scriptname or sequence of commands to execute
          (Bourne syntax)
| | | | |
| | | | |__ Weekday (0=Sun, 1=Mon, 2=Tue, etc...)
| | | |_____ Month (1 - 12)
| | |________ Day (1 - 31)
| |___________ Hour (0 - 23)
|______________ Minute (0 - 59)


Command Switches:
crontab -e	Edit your crontab file, or create one if it doesn't exist.

crontab -l	Display your crontab file.

crontab -r	Remove your crontab file.

crontab -v	Display the last time you edited your crontab file.


Crontab syntax:
	*	Wildcard, matches all values
	-	Range, matches a range of values inclusive (ie. 3 - 7 matches 3,4,5,6,7)
	,	list, matches a list of comma separated values (ie. 1,15 matches 1 and 15)
MAILTO=		Emails the output to the specified address (ie. someone@your_domain.tld)
		or leave blank for no output

    To create the initial password file for the first user:

    • cd
    • cd private
    • htpasswd -c .htpasswd username

      NOTE: The -c creates the password file called .htpasswd

    To change a password or add another user:

    • cd ~/private
    • htpasswd .htpasswd username
    • Enter the password twice when prompted.

    Note: Do not use -c to add more users, as it will create and overwrite the password file you created and wipe out your user information you first entered.

    To remove a user:

    • cd ~/private
    • pico .htpasswd
    • delete the line with the USERname you want to delete

    • How to get out of Header Hell

      Click to hide.

      In your *very* first script that loads for a given page view put
      (before anything else including html): <?php
      ob_start(); ?> In the last piece that will run, at the end put: <?php
      ob_end_flush(); ?>

      The output buffering functions will let PHP ensure that the headers go first, even if you accidently write them after starting non-header output (it buffers everything until time to send it to the client).
